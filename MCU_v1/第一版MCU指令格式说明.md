# 第一版 MCU 指令格式说明

## 文档目的

这份文档用于统一第一版 MCU 的机器码格式，服务于以下工作：

- 成员 B 编写 `decoder / controller / extend / datapath`
- 成员 A 将汇编程序翻译为机器码
- 后续将机器码写入 `instr_rom`

这份文档采用“ARM 风格裁剪版”思路：

- 保留 `32-bit` 定长指令
- 保留 `cond` 条件字段
- 保留“数据处理 / 访存 / 分支”三大类
- 只支持第一版实际需要的最小指令子集

当前第一版重点是：

- 先能写、能解码、能跑通
- 不追求完整 ARM

---

## 一、总原则

第一版统一约定：

- 指令宽度：`32-bit`
- 条件字段：`4-bit`
- 大类字段：`2-bit`
- 采用字节寻址
- 顺序执行默认 `PC = PC + 4`

第一版指令按大类分为：

```text
op[27:26] = 00  数据处理类
op[27:26] = 01  访存类
op[27:26] = 10  分支类
op[27:26] = 11  预留扩展
```

---

## 二、条件字段 cond

所有指令最高 4 位统一保留为条件字段：

```text
Instr[31:28] = cond
```

第一版正式使用：

```text
1110 = AL   Always，总是执行
0000 = EQ   Z == 1 时执行
0001 = NE   Z == 0 时执行
```

当前建议：

- 普通数据处理和访存默认使用 `AL`
- `B` 使用 `AL`
- `BEQ` 使用 `EQ`
- `BNE` 使用 `NE`

说明：

- 后续若要扩展更多条件执行，不必修改整体格式

---

## 三、第一版支持的指令

第一版支持以下指令：

```text
MOV
ADD
SUB
CMP
LDR
STR
B
BEQ
BNE
MUL
ASR
```

其中：

- `MUL` 和 `ASR` 归入数据处理类
- 访存类第一版只支持“立即数偏移”
- 分支类第一版只支持 PC 相对分支

---

## 四、数据处理类格式

### 1. 总格式

数据处理类统一格式如下：

```text
31        28 27 26 25 24    21 20 19    16 15    12 11                 0
+------------+-----+--+--------+--+--------+--------+--------------------+
|    cond    | 00  | I| opcode | S|   Rn   |   Rd   |      operand2      |
+------------+-----+--+--------+--+--------+--------+--------------------+
```

字段含义：

- `cond[31:28]`：条件执行字段
- `op[27:26] = 00`：表示数据处理类
- `I[25]`：
  - `0` = 寄存器型
  - `1` = 立即数型
- `opcode[24:21]`：操作类型
- `S[20]`：是否更新状态寄存器
- `Rn[19:16]`：第一个源寄存器
- `Rd[15:12]`：目标寄存器
- `operand2[11:0]`：第二操作数

### 2. 第一版 opcode 约定

```text
0100 = ADD
0010 = SUB
1101 = MOV
1010 = CMP
1001 = MUL
1111 = ASR
```

说明：

- `MUL` 和 `ASR` 这里属于第一版自定义扩展
- 不是要求完全兼容标准 ARM

### 3. operand2 约定

第一版不实现完整 ARM barrel shifter。

统一采用以下简化规则：

#### 当 `I = 0`（寄存器型）

```text
operand2[3:0]  = Rm
operand2[11:4] = 0
```

也就是说：

- 第二操作数来自寄存器 `Rm`
- 第一版先不支持复杂移位编码

#### 当 `I = 1`（立即数型）

```text
operand2[11:0] = imm12
```

第一版直接把 `imm12` 作为无符号立即数使用。

### 4. 各指令建议语义

#### `ADD`

寄存器型：

```text
ADD Rd, Rn, Rm
cond = AL
I    = 0
S    = 0
opcode = 0100
```

立即数型：

```text
ADD Rd, Rn, #imm12
cond = AL
I    = 1
S    = 0
opcode = 0100
```

#### `SUB`

寄存器型：

```text
SUB Rd, Rn, Rm
cond = AL
I    = 0
S    = 0
opcode = 0010
```

立即数型：

```text
SUB Rd, Rn, #imm12
cond = AL
I    = 1
S    = 0
opcode = 0010
```

#### `MOV`

寄存器型：

```text
MOV Rd, Rm
cond = AL
I    = 0
S    = 0
opcode = 1101
Rn   = 0000   ; 第一版忽略
```

立即数型：

```text
MOV Rd, #imm12
cond = AL
I    = 1
S    = 0
opcode = 1101
Rn   = 0000   ; 第一版忽略
```

#### `CMP`

```text
CMP Rn, Rm
或
CMP Rn, #imm12
```

编码要求：

- `opcode = 1010`
- `S = 1`
- 不写回普通寄存器
- `Rd` 字段第一版建议写 `0000`

说明：

- `CMP` 本质上按减法比较
- 只更新状态寄存器，不写回 `Rd`

#### `MUL`

```text
MUL Rd, Rn, Rm
cond = AL
I    = 0
S    = 0
opcode = 1001
```

约定：

- `Rn` 提供第一个乘数
- `Rm` 由 `operand2[3:0]` 指定
- `Rd` 接收结果

第一版说明：

- 第一版先不支持 `MLA`
- 第一版只取 `32-bit` 结果

#### `ASR`

第一版建议先支持立即数型：

```text
ASR Rd, Rn, #imm
cond = AL
I    = 1
S    = 0
opcode = 1111
```

约定：

- `Rn` 是被移位数据
- `imm12` 的低若干位用于移位量
- 推荐第一版只使用低 `5-bit` 作为移位量

说明：

- 这样更贴合 FFT 中常见的固定缩放，如 `ASR #15`

## 五、访存类格式

### 1. 总格式

第一版访存类统一格式如下：

```text
31        28 27 26 25 24 23         20 19    16 15    12 11             0
+------------+-----+--+--+-------------+--------+--------+---------------+
|    cond    | 01  | L| U|   reserved  |   Rn   |   Rd   |     imm12     |
+------------+-----+--+--+-------------+--------+--------+---------------+
```

字段含义：

- `cond[31:28]`：条件字段
- `op[27:26] = 01`：访存类
- `L[25]`：
  - `1` = `LDR`
  - `0` = `STR`
- `U[24]`：
  - `1` = 加偏移
  - 第一版建议固定为 `1`
- `Rn[19:16]`：基址寄存器
- `Rd[15:12]`：目标寄存器或源寄存器
- `imm12[11:0]`：立即数偏移

### 2. 第一版只支持的形式

```text
LDR Rd, [Rn, #imm12]
STR Rd, [Rn, #imm12]
```

第一版暂不支持：

- 寄存器偏移
- 负偏移
- pre/post indexing
- write-back
- 批量访存

### 3. 编码约定

#### `LDR`

```text
cond = AL
op   = 01
L    = 1
U    = 1
```

#### `STR`

```text
cond = AL
op   = 01
L    = 0
U    = 1
```

### 4. 地址语义

第一版地址计算统一为：

```text
address = Rn + zero_extend(imm12)
```

说明：

- 输入区、工作区、输出区统一按 `32-bit` 槽位访问
- 因此程序视角下地址步长统一为 `4`

---

## 六、分支类格式

### 1. 总格式

```text
31        28 27 26 25 24 23                                   0
+------------+-----+-----+--------------------------------------+
|    cond    | 10  | 00  |                imm24                 |
+------------+-----+-----+--------------------------------------+
```

字段含义：

- `cond[31:28]`：条件字段
- `op[27:26] = 10`：分支类
- `imm24[23:0]`：分支偏移

### 2. 第一版分支目标计算

第一版统一采用：

```text
branch_offset = sign_extend(imm24) << 2
pc_next       = PC + 8 + branch_offset
```

说明：

- 当前第一版实现先采用与参考 baseline 一致的 `PC + 8`
- 这样可以直接兼容现有汇编器与 `.mem` 程序文件

### 3. 对应指令

#### `B`

```text
cond = 1110   ; AL
```

#### `BEQ`

```text
cond = 0000   ; EQ
```

#### `BNE`

```text
cond = 0001   ; NE
```

---

## 七、状态寄存器更新规则

第一版建议规则如下：

### 1. 当前最保守实现

系统上先只要求：

- `CMP` 更新状态寄存器
- `BEQ/BNE` 读取 `Z`

### 2. 若后续贴近 ARM 风格

可进一步采用：

- `ADD/SUB` 在 `S=1` 时更新 `CVNZ`
- `AND/ORR` 在 `S=1` 时更新 `NZ`
- `CMP` 始终视为更新 flags 的比较指令

当前第一版建议：

- 先以 `CMP -> status_reg -> BEQ/BNE` 为主链路

---

## 八、ALUControl 建议

书中参考版本常使用 `2-bit ALUControl`，仅覆盖：

- `ADD`
- `SUB`
- `AND`
- `ORR`

但本项目第一版需要：

- `ADD`
- `SUB`
- `MOV`
- `CMP`
- `MUL`
- `ASR`

因此建议本项目采用 `3-bit ALUControl`。

建议编码如下：

```text
000 = AND
001 = ORR
010 = ADD
011 = SUB
100 = MUL
101 = ASR
110 = MOV
111 = 保留
```

说明：

- `CMP` 不必单独占用 ALUControl
- `CMP` 可直接复用 `SUB`

---

## 九、第一版实现建议顺序

建议先按以下顺序推进：

1. 先完成数据处理类最小解码：
   - `ADD`
   - `SUB`
   - `MOV`
   - `CMP`
2. 再完成访存类：
   - `LDR`
   - `STR`
3. 再完成分支类：
   - `B`
   - `BEQ`
   - `BNE`
4. 最后补：
   - `MUL`
   - `ASR`

原因：

- 先把基础 datapath 跑通
- 再逐步接入 FFT 需要的额外计算能力

---

## 十、给成员 A 的使用建议

成员 A 编写第一版汇编程序时，建议遵循：

- 访存先只写立即数偏移形式
- 分支先只使用 `B / BEQ / BNE`
- `CMP` 单独使用，不依赖其他指令顺手改 flags
- `ASR` 第一版优先写成固定移位量
- `MUL` 第一版按普通三操作数乘法使用

---

## 十一、结论

第一版机器码格式的设计原则是：

- 不追求完整 ARM
- 但保留 ARM 风格的大框架
- 让 `decoder / controller / datapath / 汇编转机器码` 都能尽快落地

当前结论：

- `MUL` 与 `ASR` 均纳入数据处理类
- 访存第一版只支持立即数偏移
- 分支第一版只统一做 PC 相对偏移
- `ALUControl` 建议扩展为 `3-bit`
